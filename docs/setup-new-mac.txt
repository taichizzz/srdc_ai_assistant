========================================================================
SeeAndSay — Build & Run on a Brand-New Mac
========================================================================
Goal: take a completely fresh macOS machine and get the SeeAndSay
Android app building and running on an Android emulator on that machine.

NOTE: The app is an Android app. It does NOT run on macOS directly.
The Mac is only the HOST that runs the Android emulator (or a plugged-in
Android phone). The "device" you test on is the emulator/phone.

This guide assumes an Apple Silicon Mac (M-series). For an Intel Mac,
see the "INTEL MAC" note at the bottom.

Run each numbered block in Terminal, in order.

------------------------------------------------------------------------
1. Command-line tools (gives you git)
------------------------------------------------------------------------
xcode-select --install

(A GUI installer pops up. Let it finish, then continue.)

------------------------------------------------------------------------
2. Homebrew, then add it to your shell
------------------------------------------------------------------------
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

------------------------------------------------------------------------
3. Java 17 + Android command-line tools
------------------------------------------------------------------------
brew install --cask temurin@17
brew install --cask android-commandlinetools

------------------------------------------------------------------------
4. Environment variables (JDK, SDK, tools on PATH)
------------------------------------------------------------------------
cat >> ~/.zshrc <<'EOF'
export JAVA_HOME="$(/usr/libexec/java_home -v 17)"
export ANDROID_HOME="/opt/homebrew/share/android-commandlinetools"
export PATH="$ANDROID_HOME/platform-tools:$ANDROID_HOME/emulator:$ANDROID_HOME/cmdline-tools/latest/bin:$PATH"
EOF
source ~/.zshrc

------------------------------------------------------------------------
5. Accept licenses + install the SDK pieces (arm64 image)
------------------------------------------------------------------------
yes | sdkmanager --licenses
sdkmanager "platform-tools" "platforms;android-34" "build-tools;34.0.0" "emulator" "system-images;android-34;google_apis;arm64-v8a"

------------------------------------------------------------------------
6. Create a phone emulator
------------------------------------------------------------------------
avdmanager create avd -n phone34 -k "system-images;android-34;google_apis;arm64-v8a" -d pixel

------------------------------------------------------------------------
7. Get the code and point Gradle at the SDK
------------------------------------------------------------------------
git clone https://github.com/taichizzz/srdc_ai_assistant.git
cd srdc_ai_assistant
echo "sdk.dir=$ANDROID_HOME" > local.properties
chmod +x gradlew

------------------------------------------------------------------------
8. Build the debug APK
------------------------------------------------------------------------
./gradlew assembleDebug

(First run downloads Gradle 8.9 + dependencies. Needs internet; a few
minutes.)

------------------------------------------------------------------------
9. Boot the emulator, wait for it, install, launch
------------------------------------------------------------------------
emulator -avd phone34 &
adb wait-for-device
until [ "$(adb shell getprop sys.boot_completed 2>/dev/null | tr -d '\r')" = "1" ]; do sleep 2; done
./gradlew installDebug
adb shell am start -n com.foxconn.seeandsay/.ui.MainActivity

This launches SeeAndSay on the emulator.

------------------------------------------------------------------------
DAY-TO-DAY REBUILDS (emulator already running)
------------------------------------------------------------------------
./gradlew installDebug

------------------------------------------------------------------------
PHASE 3 (MICROPHONE) TESTING
------------------------------------------------------------------------
The emulator can use the Mac's built-in mic, but it is OFF by default.
Enable it in the emulator window:
    ... (Extended controls) -> Microphone
    -> "Virtual microphone uses host audio input"

For the most reliable mic/speaker test, plug in a real Android phone
(enable USB debugging). Then:
    adb devices          # confirm the phone is listed
    ./gradlew installDebug   # installs to the phone

------------------------------------------------------------------------
INTEL MAC (x86_64) NOTES
------------------------------------------------------------------------
- Homebrew lives at /usr/local, so set:
    export ANDROID_HOME="/usr/local/share/android-commandlinetools"
- Use the x86_64 system image everywhere instead of arm64-v8a:
    system-images;android-34;google_apis;x86_64

------------------------------------------------------------------------
EASIER ALTERNATIVE (GUI SDK setup)
------------------------------------------------------------------------
If the terminal SDK route gives trouble:
    brew install --cask android-studio
Open Android Studio once, let its setup wizard install the SDK and
create an AVD, then steps 7-9 above still work from the terminal.
Android Studio just replaces steps 3, 5, and 6.

------------------------------------------------------------------------
COMMON PROBLEMS
------------------------------------------------------------------------
- "sdkmanager: command not found": step 4 PATH not applied. Run
  `source ~/.zshrc` or open a new Terminal tab.
- Gradle can't find the SDK: check local.properties has the correct
  sdk.dir (must match $ANDROID_HOME).
- Wrong ANDROID_HOME: run `echo $ANDROID_HOME` and confirm the folder
  exists (`ls "$ANDROID_HOME"`).
