# M1.3 — Voice Interaction Loop Acceptance

This script verifies the Week 1 mainline loop:

`final transcript → RuleBasedReplyEngine → composed cloud/device TTS → Idle`

The app remains user-initiated push-to-talk. The user presses **Start** once and speaks; the first
non-blank final result from production STT now closes microphone input automatically through the
same bounded drain path as manual **Stop**. Reply decision and playback then happen automatically.
Stop remains an idempotent override when recognition is slow or the user wants to end early. The app
never adds a wake word, listens before Start, reopens the mic after speaking, or permits Start while
the assistant is replying or speaking.

This behavior is provider-final endpointing, not a new local VAD model: automatic completion depends
on Google returning a non-blank final `SttResult`. If that signal is delayed, press **Stop**; the
manual path remains fully supported and uses the same microphone-release/cloud-drain sequence.

## Automated gate — no device, credential, or network

From the repository root, run:

```sh
./gradlew testDebugUnitTest assembleDebug lintDebug assembleRelease
```

Expected result: all unit tests pass, Android lint has no blocking issue, and both debug and release
APKs assemble. The fake-based M1.3 tests prove:

- interim transcripts never generate speech;
- interim transcripts keep the user-started session in `Listening`, while the first non-blank final
  automatically requests microphone shutdown without a second button press;
- all non-blank final segments from one STT stream are joined in order and generate exactly one
  reply/TTS request after the stream completes;
- repeated finals and a manual Stop/final race share one idempotent end request;
- typed input uses the same final-transcript reducer and reply/TTS path;
- microphone capture is cancelled and joined before TTS begins;
- reply/TTS failure reaches recoverable `Error`, and **Retry** restores `Idle`;
- cancellation stops speech without becoming a user-visible error;
- disabling **Automatic voice reply** retains the transcript without speaking; and
- a second explicit Start works after the first loop returns to `Idle`—there is no auto re-listen.

The pure reply-engine tests also cover NFKC normalization, punctuation/whitespace removal, local
intent synonyms, open/play target extraction, missing-target clarification, helpful unknown input,
repeat/cancel context, bounded memory, explicit verified-action recording, and non-blank replies.

## Mic-less emulator path

Requirements: a debug build. No microphone permission, STT credential, or network is required. An
installed Taiwan-Mandarin device TTS voice is required for audible fallback; otherwise the visible
`Replying`/`Speaking` transition and recoverable TTS state can still be inspected.

1. Launch **IVI AI 助理** and leave **Automatic voice reply** enabled (the default).
2. At the top of the screen, enter `你好` in **Typed transcript** and press **Submit transcript**.
3. Confirm **Accumulated final transcript** shows `你好`.
4. Confirm **Assistant reply (current/last)** shows
   `你好，我是AI 助理 Roxanne，很高興為你服務。`.
5. Confirm the app enters `Replying`, then `Speaking`, then `Idle` after playback. With usable cloud
   or device TTS, hear the greeting without pressing the standalone DEBUG TTS **Speak** button.
6. Turn **Automatic voice reply** off, submit `你好` again, and confirm the transcript is appended
   but no new reply is spoken. The session remains `Completed`, preserving transcript-only testing.
7. Turn the toggle back on for the real-device checks.

### Automatic-reply TTS model selector

This DEBUG selector controls the TTS client inside the main
`final transcript → ReplyEngine → TtsClient` pipeline. It is separate from the standalone M1.2 TTS
selector farther down the screen. Release builds expose no selector and retain WaveNet.

1. Under **Automatic voice reply**, select **WaveNet (`cmn-TW-Wavenet-B`)**.
2. Submit typed `你好`; confirm the normal greeting reply is generated and spoken once.
3. Select **Gemini TTS (Kore)** and confirm the selected line shows
   `gemini-3.1-flash-tts-preview` / `Kore`.
4. Submit typed `你好` again; confirm the same deterministic reply now routes through the Gemini
   cloud profile using the configured Taiwan-Mandarin driving prompt.
5. While STT, Replying, Speaking, loopback, cloud smoke testing, or standalone TTS is active, confirm
   both main-pipeline model controls are disabled and the in-flight utterance does not switch.
6. Check Logcat tag `TtsModel` for `pipeline_model=wavenet` or `pipeline_model=gemini`, and tag
   `TtsEngine` for the actual `engine=cloud` / `engine=on_device` route. A selected Gemini label does
   not prove Google accepted synthesis: cloud failure still invokes the existing device fallback.

Gemini live acceptance requires a current short-lived bearer token whose service account has
`aiplatform.endpoints.predict`; WaveNet can continue using the restricted API key. Typed selection
and routing state are covered by unit tests, while audible model identity requires a device/network.

### Local intents and conversation memory

These checks require only typed input plus an available cloud/device TTS route. They do not require
a microphone, STT credential, AccessibilityService, LM, or screen-operation implementation.

1. Submit `你可以做什麼？` and confirm the assistant lists examples including `打開設定`,
   `再說一次`, and `取消`.
2. Submit `請幫我打開設定！` and confirm the assistant says
   `收到，你想開啟「設定」。目前尚未連接畫面操作，請稍後再試。`. This intentionally
   acknowledges the parsed target without claiming Settings opened; actual screen operation starts
   in M2.2 and verified success starts in M2.3.
3. Submit `再說一次` and confirm the immediately preceding assistant reply is spoken again.
4. Submit `取消` and confirm `已取消目前記住的操作要求。`. Submit `取消` once more and confirm
   `目前沒有等待中的操作要求。`. This clears conversational target context only; it is not a
   barge-in or coroutine-cancellation command.
5. Submit `打開` and confirm the assistant asks which function to open/play, with concrete examples.
6. Submit `今天天氣如何？` and confirm the fallback preserves the transcript and suggests saying
   `幫助`; it must not claim general knowledge or invoke an LM.

Conversation context is intentionally lightweight: it retains at most the last transcript, reply,
requested target, and future verified action in bounded process memory. It is cleared when the app
process is recreated, is never persisted or logged, and cannot yet contain a successful screen
action because M2.2/M2.3 have not executed and verified one.

The transcript and assistant reply must remain directly below session status and Start/Stop at the
top of a phone-sized screen. Cloud configuration, STT engines/metrics, loopback, and standalone TTS
controls remain farther down the scrollable debug page.

## Real-device microphone path

Requirements: a real microphone/speaker device, granted `RECORD_AUDIO`, network access, and a live
credential accepted by the production Google STT V1 path. Cloud TTS may reuse the configured API
key/token; if cloud synthesis is disabled or unavailable, installed `zh-TW` device TTS is the
expected fallback. Credential values stay only in gitignored `local.properties`/the approved token
workflow and must never be committed or logged.

### Greeting

1. Ensure **Automatic voice reply** is enabled and the session is `Idle`.
2. Press **Start**, say exactly `你好`, and observe the live partial transcript at the top.
3. Do **not** press Stop. Confirm the first final `你好` automatically moves the session through
   `Stopping`; this is the explicit final-result endpointing behavior under test.
4. Confirm `你好` appears in **Accumulated final transcript** and the microphone is released.
5. Without pressing Stop or any TTS control, confirm **Assistant reply (current/last)** shows and audibly
   speaks `你好，我是AI 助理 Roxanne，很高興為你服務。`.
6. Confirm state returns to `Idle` and the microphone indicator is no longer active before playback.

### Helpful fallback and next session

1. Press **Start** again, say exactly `請播放我最喜歡的歌`, and wait for the provider final without
   pressing Stop.
2. Confirm the new final transcript is appended and the assistant automatically speaks
   `收到，你想播放「我最喜歡的歌」。目前尚未連接畫面操作，請稍後再試。` exactly once.
3. Confirm the app returns to `Idle` and does not listen again until another explicit Start.

### Manual Stop override

1. Press **Start**, begin a longer phrase, and press **Stop** before a provider final appears.
2. Confirm the app enters `Stopping`, releases the microphone, drains any final result, and produces
   at most one reply for the completed session.
3. Repeat while a final is arriving and confirm the Stop/final race does not duplicate transcript
   text, reply generation, or TTS playback.

## Recovery checks

- If production STT fails, confirm `Error` is visible and **Retry** restores a usable `Idle` state.
- If a final result arrives but the cloud stream does not finish draining within the bound, confirm
  the session reaches recoverable `Error` rather than hanging in `Stopping`; **Retry** restores Idle.
- If both cloud and on-device TTS fail, confirm the reply remains visible, `Error` is recoverable,
  and the microphone can be started again after **Retry**.
- While integrated state is `Replying` or `Speaking`, confirm Start, loopback, cloud STT test, typed
  submission, and standalone TTS controls cannot start competing audio work.

The automated suite deterministically proves orchestration, exactly-once behavior, echo ordering,
toggle behavior, cancellation, and recovery. Actual Mandarin recognition, audible pitch/voice,
cloud credential acceptance, audio focus, and hardware microphone release require the real-device
steps above.

## Milestone result

M1.3 is code-complete when both the automated gate and applicable manual checks pass: after one
explicit Start, saying `你好` produces a provider final that automatically stops capture, commits
the transcript, chooses the preset reply, and speaks it without manual Stop, reply, or TTS action.
Typed `你好` remains the mic-less path. Real-device/cloud endpoint timing and audible acceptance
remain explicit human verification steps.
