# M1.3 — Voice Interaction Loop Acceptance

This script verifies the Week 1 mainline loop:

`final transcript → RuleBasedReplyEngine → composed cloud/device TTS → Idle`

The app remains push-to-talk. The user presses **Start**, speaks, and presses **Stop** to close the
utterance; the reply decision and playback then happen automatically. The app never opens the mic
again after speaking, and Start remains unavailable while the assistant is replying or speaking.

## Automated gate — no device, credential, or network

From the repository root, run:

```sh
./gradlew testDebugUnitTest assembleDebug lintDebug assembleRelease
```

Expected result: all unit tests pass, Android lint has no blocking issue, and both debug and release
APKs assemble. The fake-based M1.3 tests prove:

- interim transcripts never generate speech;
- all non-blank final segments from one STT stream are joined in order and generate exactly one
  reply/TTS request after the stream completes;
- typed input uses the same final-transcript reducer and reply/TTS path;
- microphone capture is cancelled and joined before TTS begins;
- reply/TTS failure reaches recoverable `Error`, and **Retry** restores `Idle`;
- cancellation stops speech without becoming a user-visible error;
- disabling **Automatic voice reply** retains the transcript without speaking; and
- a second explicit Start works after the first loop returns to `Idle`—there is no auto re-listen.

The pure reply-engine tests also cover NFKC normalization, punctuation/whitespace removal, greeting
variants, unmatched fallback, and the non-blank reply guarantee.

## Mic-less emulator path

Requirements: a debug build. No microphone permission, STT credential, or network is required. An
installed Taiwan-Mandarin device TTS voice is required for audible fallback; otherwise the visible
`Replying`/`Speaking` transition and recoverable TTS state can still be inspected.

1. Launch **IVI AI 助理** and leave **Automatic voice reply** enabled (the default).
2. At the top of the screen, enter `你好` in **Typed transcript** and press **Submit transcript**.
3. Confirm **Accumulated final transcript** shows `你好`.
4. Confirm **Assistant reply (current/last)** shows
   `你好，我是 IVI AI 助理，很高興為你服務。`.
5. Confirm the app enters `Replying`, then `Speaking`, then `Idle` after playback. With usable cloud
   or device TTS, hear the greeting without pressing the standalone DEBUG TTS **Speak** button.
6. Turn **Automatic voice reply** off, submit `你好` again, and confirm the transcript is appended
   but no new reply is spoken. The session remains `Completed`, preserving transcript-only testing.
7. Turn the toggle back on for the real-device checks.

The transcript and assistant reply must remain directly below session status and Start/Stop at the
top of a phone-sized screen. Cloud configuration, STT engines/metrics, loopback, and standalone TTS
controls remain farther down the scrollable debug page.

## Real-device microphone path

Requirements: a real microphone/speaker device, granted `RECORD_AUDIO`, network access, and a live
credential accepted by the production Google STT V1 path. Cloud TTS may reuse the configured API
key/token; if cloud synthesis is disabled or unavailable, installed `zh-TW` device TTS is the
expected fallback. Credential values stay only in gitignored `local.properties`/the approved token
workflow and must never be committed or logged.

### Greeting

1. Ensure **Automatic voice reply** is enabled and the session is `Idle`.
2. Press **Start**, say exactly `你好`, and observe the live partial transcript at the top.
3. Press **Stop** to end this push-to-talk utterance.
4. Confirm the final `你好` appears in **Accumulated final transcript**.
5. Without pressing any TTS control, confirm **Assistant reply (current/last)** shows and audibly
   speaks `你好，我是 IVI AI 助理，很高興為你服務。`.
6. Confirm state returns to `Idle` and the microphone indicator is no longer active before playback.

### Unmatched fallback and next session

1. Press **Start** again, say exactly `請播放我最喜歡的歌`, then press **Stop**.
2. Confirm the new final transcript is appended and the assistant automatically speaks
   `我聽到了：『請播放我最喜歡的歌』，但我還不知道怎麼回應。` exactly once.
3. Confirm the app returns to `Idle` and does not listen again until another explicit Start.

## Recovery checks

- If production STT fails, confirm `Error` is visible and **Retry** restores a usable `Idle` state.
- If both cloud and on-device TTS fail, confirm the reply remains visible, `Error` is recoverable,
  and the microphone can be started again after **Retry**.
- While integrated state is `Replying` or `Speaking`, confirm Start, loopback, cloud STT test, typed
  submission, and standalone TTS controls cannot start competing audio work.

The automated suite deterministically proves orchestration, exactly-once behavior, echo ordering,
toggle behavior, cancellation, and recovery. Actual Mandarin recognition, audible pitch/voice,
cloud credential acceptance, audio focus, and hardware microphone release require the real-device
steps above.

## Milestone result

M1.3 is code-complete when both the automated gate and applicable manual checks pass: saying or
typing `你好` commits the transcript, chooses the preset reply, and speaks it with no manual reply
or TTS action. Real-device/cloud acceptance remains an explicit human verification step.
