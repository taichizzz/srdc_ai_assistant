# M1.1 — Streaming Speech-to-Text Demo

## Purpose

Verify that the AAOS application captures Taiwan Mandarin, streams it to Google
Cloud Speech-to-Text, displays interim text, and preserves the final transcript.

## Current implementation status

Phase 2 provides the buildable debug UI, runtime microphone permission recovery,
push-to-talk state transitions, and permission-independent typed transcript
injection. Audio capture and cloud recognition remain future M1.1 phases.

## Prerequisites

1. Install the debug APK on an API 29+ Android or AAOS device.
2. Ensure the device has network access and a working microphone.
3. An OAuth access token is not required until the cloud-client phase.
4. Never place a service-account JSON file in the project or APK.

## Phase 1 foundation checkpoint (historical)

1. Build and install the debug application.
2. Launch **SeeAndSay** from the device launcher.
3. Confirm the screen shows **M1.1 Android foundation is ready**.
4. Confirm the app remains responsive and does not request microphone permission yet.

Verified on 2026-07-15 with an API 29 phone AVD: the APK installed, launched cold,
reached the resumed state, and exposed both expected strings in the UI hierarchy.
An AAOS AVD was not configured on the development machine, so AAOS-specific visual
verification remains a later device check; Phase 1 uses no automotive-only API.

## Phase 2 permission and typed-input checkpoint

### Typed input without microphone permission

1. Clear the application's data or start with microphone permission not requested.
2. Launch **SeeAndSay**.
3. Confirm **Microphone permission: Not requested** and **Session status: Idle**.
4. Do not press **Start** and do not grant microphone permission.
5. Enter exactly **「顯示」** in **Transcript text**.
6. Press **Submit transcript**.
7. Confirm **「顯示」** appears exactly once under **Accumulated final transcript**.
8. Confirm the partial transcript is empty and the session status is **Completed**.

### Granted permission and push-to-talk state

1. Press **Start**.
2. Grant the microphone permission in Android's dialog.
3. Confirm **Microphone permission: Granted**, **Session status: Listening**, and the
   button label **Stop**.
4. Confirm no transcript text was fabricated by pressing Start.
5. Press **Stop**.
6. Confirm **Session status: Completed** and the button label returns to **Start**.

### Retryable denial

1. Reset/revoke microphone permission and press **Start**.
2. Press **Deny** in Android's permission dialog.
3. Confirm **Microphone permission: Denied — retry available** and
   **Session status: Error**.
4. Confirm the rationale text and **Retry** action are visible.
5. Press **Retry** and confirm the screen returns to **Idle** without crashing.
6. Press **Start** and confirm Android can present the permission request again.

### Permanent denial

1. In Android's repeated permission dialog, select **Deny & don’t ask again**.
2. Confirm **Microphone permission: Permanently denied — open settings** and
   **Session status: Error**.
3. Confirm **Retry** and **Open settings** are both visible.
4. Press **Open settings** and confirm Android opens SeeAndSay's application-details page.
5. Grant microphone permission in Settings and return to the app.
6. Confirm permission changes to **Granted** and the session returns to a usable
   **Idle** state without starting automatically.

Verified on 2026-07-15 with an API 29 phone AVD. Typed input committed while
permission remained not requested; retryable denial, permanent denial, Settings
recovery, grant, Start, and Stop all completed without a crash. The automated ADB
input used ASCII text while the pure ViewModel test verified Traditional Chinese
text through the same reducer.

## Final M1.1 acceptance script

> The permission, controls, state UI, and typed-input behavior are implemented.
> Microphone capture and Google Cloud results remain subsequent M1.1 phases.

1. Launch **SeeAndSay**.
2. Press **Start** and grant microphone permission when prompted.
3. Say exactly: **「你好」**.
4. While speaking, confirm a live partial transcript appears and is replaced as the
   recognizer revises it.
5. Press **Stop**.
6. Confirm the final transcript contains **「你好」** and the partial line clears.
7. Start a second session and say exactly: **「打開設定」**.
8. Confirm the final transcript contains **「打開設定」** exactly once.
9. Enter **「顯示」** in the typed-input debug field and submit it.
10. Confirm **「顯示」** enters the same final-transcript path without microphone use.

## Recovery checks

1. Deny microphone permission and confirm a recoverable permission message appears.
2. Disable networking during a session and confirm the UI leaves its busy state with a
   recoverable network message.
3. Use an expired OAuth token and confirm authentication rejection is visible.
4. Stop a session and immediately start another; confirm the prior microphone and stream
   have been released.
